# Public Access Tokens (PAT)

## What They Are

Short-lived JWTs scoped to a single task. Safe to send to the browser — cannot access other tasks or endpoints.

## How They Work

1. **Server**: start a task with `generateAsync()`, receive a PAT
2. **Client**: use the PAT to poll task status and results directly from the browser
3. CORS is enabled on `/tasks/:id/status` and `/tasks/:id/results` — no proxy needed

## Server-Side: Generate and Send to Client

```ts
import { mynth } from "./mynth";

const taskAsync = await mynth.image.generateAsync({ prompt: "A sunset" });

// Send both to the client
const taskId = taskAsync.id;
const pat = taskAsync.access.publicAccessToken;
```

PATs are auto-generated by default. Disable with `access.pat.enabled: false` in the generate request.

## Client-Side: Poll from Browser

```ts
const status = await fetch(`https://api.mynth.io/tasks/${taskId}/status`, {
  headers: { Authorization: `Bearer ${pat}` },
}).then((r) => r.json());

if (status.status === "completed") {
  const results = await fetch(`https://api.mynth.io/tasks/${taskId}/results`, {
    headers: { Authorization: `Bearer ${pat}` },
  }).then((r) => r.json());

  console.log(results.images);
}
```

## Safety

- PATs are scoped to a single task ID only
- Cannot access `GET /tasks/:id` (full details — API key only)
- Cannot access other tasks
- CORS allows all origins on status/results endpoints
- Short-lived (1 hour when auto-generated)

## SDK Generate Async

The SDK handles PAT generation and polling internally:

```ts
const taskAsync = await mynth.image.generateAsync({ prompt: "..." });
console.log(taskAsync.access.publicAccessToken); // send this to client
const task = await taskAsync.wait(); // polls until done
```
